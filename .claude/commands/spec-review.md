# /spec-review — 설계서 기획 검토

---

## 실행 방법

**채팅 입력창에 입력:**
```
/spec-review
```

설계서(또는 요약본)를 텍스트로 붙여넣거나 파일 경로를 알려주면
기획에서 추가 확인이 필요한 항목을 정리하고, 프로젝트 폴더에 `spec-gaps.md`로 저장한다.

# 이하 Claude 실행 지침 — `/spec-review` 입력 시 아래 순서대로 진행한다
---
---

## STEP 0. 입력 수집

아래 항목을 질문한다. 이미 제공된 항목은 건너뛴다.

1. **설계서 내용** — 텍스트 붙여넣기 또는 파일 경로
2. **도메인/화면 범위** — 전체인지 특정 화면인지 (없으면 전체로 간주)
3. **프로젝트 컨텍스트** — 신규인지 기존 운영 중인지 (없으면 신규로 간주)
4. **프로젝트 경로** — `spec-gaps.md`를 저장할 프로젝트 루트 폴더
   - 기본값: 현재 작업 디렉토리 안의 `projects/{PROJECT}/`
   - 다른 위치면 절대 경로 지정 (예: `C:\Users\user\Desktop\eKYC`)
   - **이 경로가 `{ROOT_PATH}`가 된다.**

---

## STEP 1. 설계서 분석

입력된 설계서를 아래 8개 관점으로 검토한다.
각 관점별로 문제가 없으면 건너뛰고, 이슈가 있는 항목만 출력한다.

| # | 관점 | 확인 내용 |
|---|------|----------|
| 1 | 상태 누락 | 에러 / 빈값 / 로딩 / 성공 / 완료 중 정의되지 않은 상태 |
| 2 | 분기 조건 불명확 | "경우에 따라", "필요 시" 등 기준이 없는 조건 분기 |
| 3 | 화면 흐름 단절 | 이동 대상 화면이 정의되지 않은 경우 |
| 4 | 컴포넌트 미정의 | 팝업 / 바텀시트 / 토스트 등 언급만 있고 내용 없는 것 |
| 5 | 엣지 케이스 | 재시도 횟수 / 타임아웃 / 중복 요청 / 네트워크 오류 처리 기준 |
| 6 | 사용자 상태 분기 | 비회원 / 회원 / 만료 / 권한 없음 등 케이스별 처리 누락 |
| 7 | 문구 미확정 | 버튼명 / 레이블 / 안내 문구가 TBD이거나 비어 있는 것 |
| 8 | 데이터 바인딩 | 어떤 값을 어디서 받아오는지 명시되지 않은 필드 |

---

## STEP 2. 결과 출력

아래 형식으로 출력한다.

```
## 기획 확인 필요 항목 — {도메인/화면명}

### 🔴 필수 확인 (HTML 작업 전 반드시 결정)
- [화면명] {이슈 내용}
  → 확인 필요: {기획팀에 물어볼 질문}

### 🟡 작업 중 확인 (가정하고 진행 가능하나 확인 권장)
- [화면명] {이슈 내용}
  → 확인 필요: {기획팀에 물어볼 질문}

### 🟢 참고 (영향 낮음, 추후 정리)
- [화면명] {이슈 내용}
```

**우선순위 기준:**
- 🔴 필수: 해당 항목 없이 화면 구현 자체가 불가능한 것
- 🟡 작업 중: 가정하고 진행할 수 있지만 나중에 수정 가능성 있는 것
- 🟢 참고: 구현에 영향 없지만 완성도를 위해 확인하면 좋은 것

---

## STEP 3. 요약

전체 이슈 수와 가장 중요한 항목 1~2개를 한 줄로 요약한다.

```
총 {N}건 발견 — 🔴 {n}건 / 🟡 {n}건 / 🟢 {n}건
가장 먼저 확인할 것: {항목}
```

---

## STEP 4. spec-gaps.md 저장

분석 결과를 `{ROOT_PATH}/spec-gaps.md`에 저장한다.

- 파일이 없으면 새로 생성한다.
- 파일이 이미 있으면 **기존 내용은 유지**하고 새 섹션을 **맨 아래에 추가**한다.

### spec-gaps.md 형식

```markdown
# Spec Gaps — {프로젝트명}

> 마지막 업데이트: {날짜}

---

## {설계서명 또는 도메인} — {분석 날짜}

| 상태 | 우선순위 | 화면 | 내용 | 확인 필요 사항 |
|------|---------|------|------|--------------|
| ⬜ 미확인 | 🔴 | [화면명] | {이슈 내용} | {기획팀 질문} |
| ⬜ 미확인 | 🟡 | [화면명] | {이슈 내용} | {기획팀 질문} |
| ⬜ 미확인 | 🟢 | [화면명] | {이슈 내용} | — |
```

**상태 값:**
- `⬜ 미확인` — 아직 확인 안 됨 (기본값)
- `🔄 확인중` — 기획자에게 질문 중
- `✅ 완료` — 답변 받음, 내용 확정
- `⏸ 보류` — 당장 작업 범위 밖, 나중에 처리

저장 후 파일 경로를 채팅에 출력한다.
