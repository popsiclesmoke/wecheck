### QA-TERM-001 화면 진입 및 초기 상태 검증

**목적**
> MAIN 도메인에서 TERM-001로 진입했을 때 초기 UI 상태가 design-spec.md 기준과 일치하는지 확인한다.

**사전 조건**
- MAIN-001~004 공지 확인 절차 완료 상태
- TERM-001(`A02.위체크메인_약관동의.html`) 화면 진입

**테스트 단계**

| 단계 | 액션 | 기대 결과 |
|------|------|----------|
| 1 | TERM-001 화면 로드 | `StatusBar`(44px) + `MainHeader`(50px, 뒤로가기 + 로고) 표시 |
| 2 | 본문 타이틀 확인 | "약관에 동의해주세요" 텍스트가 `display1` 22px로 표시, `padding: 30px 20px 40px` 적용 |
| 3 | 전체동의 체크박스 확인 | `Checkbox size="M"`(19×19px) + "전체 동의" 레이블 표시, 구분선 존재, 초기 상태 미체크 |
| 4 | 개별 약관 항목 3종 확인 | `Checkbox size="S"`(16×16px) + [필수] 빨간 점 마커 + 약관명 + chevron(>) 각각 표시, 행 높이 40~42px |
| 5 | 하단 버튼 확인 | "동의하고 계속하기" 버튼이 `disabled` 상태로 표시 (탭 불가) |

### QA-TERM-002 전체 동의 체크 — 버튼 활성화

**목적**
> "전체 동의" 체크박스 탭 시 모든 항목이 일괄 체크되고 버튼이 `primary`로 전환되는지 확인한다.

**사전 조건**
- TERM-001 화면 초기 상태 (모든 항목 미체크, 버튼 `disabled`)

**테스트 단계**

| 단계 | 액션 | 기대 결과 |
|------|------|----------|
| 1 | `Checkbox size="M"` (전체 동의) 탭 | 전체 동의 체크박스가 체크(선택) 상태로 전환 |
| 2 | 개별 항목 3종 상태 확인 | 서비스 이용약관 / 개인(신용)정보 수집·이용 / 개인(신용)정보 제3자 제공 체크박스 모두 체크 상태로 일괄 전환 |
| 3 | 하단 버튼 상태 확인 | "동의하고 계속하기" 버튼이 `variant="primary"`(활성) 상태로 전환 |

**예외 케이스**

| 케이스 | 액션 | 기대 결과 |
|--------|------|----------|
| 재탭(토글) | 전체 동의 체크박스를 다시 탭 | 전체 항목 일괄 해제, 버튼 `disabled`로 복귀 |

**결과**
- [ ] 통과
- [x] 실패 — 사유: JavaScript 인터랙션 로직 미구현. `A02.위체크메인_약관동의.html`에 체크박스 클릭 이벤트 핸들러가 없어 전체동의 탭 시 개별 항목 일괄 체크 및 버튼 상태 전환이 동작하지 않음.
