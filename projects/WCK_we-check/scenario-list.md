## 시나리오 상세

### SCN-WCK-001: 증명서 발급 전체 여정 (앱 최초 진입 → 발급 완료)
**분류:** Happy Path
**설명:** 앱을 처음 실행한 사용자가 공지 확인·약관 동의·본인인증을 거쳐 주소를 입력하고 결과를 선택한 뒤, 인증번호 입력·서류 업로드로 발급을 완료하고 증명서 결과를 조회한다. (서비스 핵심 E2E 흐름)
**화면 흐름:**
```
MAIN-001: 위체크 메인 (앱 최초 진입)
→ MAIN-004: 공지팝업 (2개 이상)
→ TERM-001: 약관 동의 (필수 약관 전체 동의)
→ AUTH-001: 인증 방식 선택
→ AUTH-002: 간편인증서 선택
→ AUTH-005: 간편인증서 목록 로딩
→ AUTH-006: PASS 통신사 선택 (팝업)
→ AUTH-008: 인증 요청
→ AUTH-009: 인증 요청 (대기 중)
→ AUTH-011: 인증번호 입력
→ RSLT-001: 인증 완료 (=AUTH-013)
→ SRCH-001: 주소 정보 입력
→ SRCH-003: 주소 정보 입력 (활성화)
→ SRCH-005: 검색 결과 목록 선택
→ ISSU-001: 인증번호 입력
→ ISSU-003: 서류 업로드
→ RSLT-014: 대기열 로딩
→ RSLT-004: 인증 완료 — 증명서 결과 조회
```

### SCN-WCK-002: 공지 확인 후 약관 동의 (공지 2건 이상)
**분류:** Happy Path
**설명:** 앱 진입 시 노출되는 공지팝업(2건 이상, swipe + dot)을 확인하고 약관 동의 단계로 진입한다.
**화면 흐름:**
```
MAIN-001: 위체크 메인 (앱 최초 진입)
→ MAIN-004: 공지팝업 (2개 이상, swipe + dot)
→ MAIN-001: 위체크 메인 (공지 확인 후 복귀)
→ TERM-001: 약관 동의
```
> 비고: 공지 1건일 경우 MAIN-002(짧음)/MAIN-003(스크롤)로 조건 분기. 공지가 없으면 MAIN-001에서 바로 TERM-001로 이동.

### SCN-WCK-004: 본인인증 (PASS · 인증번호 방식)
**분류:** Happy Path
**설명:** 간편인증서 + PASS 통신사 선택 후 인증번호 입력 방식으로 본인인증을 완료하고 주소 입력 단계로 진입한다.
**화면 흐름:**
```
MAIN-001: 위체크 메인 (앱 최초 진입)
→ TERM-001: 약관 동의
→ AUTH-001: 인증 방식 선택
→ AUTH-002: 간편인증서 선택
→ AUTH-005: 간편인증서 목록 로딩
→ AUTH-006: PASS 통신사 선택 (팝업)
→ AUTH-008: 인증 요청
→ AUTH-009: 인증 요청 (대기 중)
→ AUTH-011: 인증번호 입력
→ RSLT-001: 인증 완료
→ SRCH-001: 주소 정보 입력
```
> 변형: 간편인증서 선택 화면은 장차법(AUTH-003)·1열(AUTH-004) 레이아웃 변형 존재.

### SCN-WCK-005: 주소 입력 → 결과 선택 (기본)

**화면 흐름:**
```
RSLT-001: 인증 완료
→ SRCH-001: 주소 정보 입력 (다음 버튼 비활성)
→ SRCH-003: 주소 정보 입력 (활성화)
→ SRCH-005: 검색 결과 목록 선택 (라디오 단일 선택)
→ ISSU-001: 인증번호 입력
```
> 진입 경로: MAIN-001(앱 최초 진입) → … → AUTH-011 → RSLT-001 → SRCH-001

> 비고: 취소 시 SRCH-005에서 기존 선택 유지.

### SCN-WCK-108: 정보 조회 오류 (유형별 안내)
**분류:** 분기
**설명:** 주소·정보 조회 중 실패가 발생하면 유형별(비회원·생년월일·세션·실패) 오류 화면으로 안내하고 재시도 또는 메인 복귀를 유도한다.
**화면 흐름:**
```
SRCH-001: 주소 정보 입력 (조회 요청)
→ RSLT-015: 에러 조회 실패 팝업 (오버레이)
→ RSLT-007: 에러 — 정보 조회 (비회원)   ※ 유형별: RSLT-008 생년월일 / RSLT-009 세션 / RSLT-010 실패
→ RSLT-016: 에러 — 2버튼 (재시도 / MAIN 복귀)
```
> 비고: 주소 처리 오류 시 RSLT-013(에러 — 주소), 네트워크 오류 시 RSLT-005, 분류 불가 시 RSLT-006로 분기.

```
SRCH-005: 검색 결과 목록 선택 (세션 만료 발생)

```
> 비고: AUTH 구간 세션 만료(AUTH-014)와 화면은 같으나 트리거 도메인이 다름. CR-002/CR-003 연계.

### SCN-WCK-110: 인증서 미보유 / 인증서 오류 안내
**분류:** 분기

> 진입 경로: MAIN-001 → TERM-001 → AUTH-001 → AUTH-002
