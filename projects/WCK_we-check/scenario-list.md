## 시나리오 상세

### SCN-WCK-001: 증명서 발급 전체 여정 (앱 최초 진입 → 발급 완료 → 이력 확인)
**분류:** Happy Path
**사용자 목적:** 앱을 처음 실행한 사용자가 공지 확인·약관 동의·본인인증을 거쳐 원하는 증명서를 발급받고, 발급 이력까지 확인한다. (서비스 핵심 E2E 흐름)
**화면 흐름:**
```
MAIN-001: 위체크 메인 (앱 최초 진입)
  → MAIN-004: 공지팝업 (2개 이상)
    → TERM-001: 약관 동의 (필수 약관 전체 동의)
      → AUTH-001: 인증 방식 선택
        → AUTH-002: 간편인증서 선택
          → AUTH-005: 간편인증서 목록 로딩
            → AUTH-006: PASS 통신사 선택 (팝업)
              → AUTH-008: 인증 요청
                → AUTH-009: 인증 요청 (대기 중)
                  → AUTH-011: 인증번호 입력
                    → AUTH-013: 인증 완료
                      → SRCH-001: 기관 검색·선택
                        → SRCH-002: 증명서 종류 선택
                          → ISSU-001: 발급 정보 입력
                            → ISSU-002: 발급 정보 확인
                              → ISSU-003: 발급 처리 중
                                → ISSU-004: 발급 완료
                                  → HIST-001: 발급 이력 목록
```

---

### SCN-WCK-002: 공지 확인 후 약관 동의 (공지 2건 이상)
**분류:** Happy Path
**사용자 목적:** 앱 진입 시 노출되는 공지팝업(2건 이상, swipe)을 확인하고 약관 동의 단계로 진입한다.
**화면 흐름:**
```
MAIN-001: 위체크 메인 (앱 최초 진입)
  → MAIN-004: 공지팝업 (2개 이상, swipe + dot)
    → (확인) → MAIN-001: 위체크 메인
      → TERM-001: 약관 동의
```
> 비고: 공지 1건일 경우 MAIN-002(짧음)/MAIN-003(스크롤)로 분기. 공지가 없으면 MAIN-001에서 바로 TERM-001로 이동.

---

### SCN-WCK-003: 약관 상세 확인 후 동의
**분류:** Happy Path
**사용자 목적:** 약관 동의 화면에서 각 약관 항목 상세 내용을 바텀시트로 확인한 뒤 전체 동의하고 인증 단계로 진입한다.
**화면 흐름:**
```
TERM-001: 약관 동의 (항목 탭)
  → TERM-002: 서비스 이용약관 상세 (바텀시트)
    → TERM-003: 개인(신용)정보 수집·이용 동의 상세 (바텀시트)
      → TERM-004: 개인(신용)정보 제3자 제공 동의 상세 (바텀시트)
        → TERM-001: 약관 동의 (전체 동의 후)
          → AUTH-001: 인증 방식 선택
```
> 진입 경로: MAIN-001(앱 최초 진입) → 공지 확인 후 → TERM-001

---

### SCN-WCK-004: 본인인증 (PASS · 인증번호 방식)
**분류:** Happy Path
**사용자 목적:** 간편인증서 + PASS 통신사 선택 후 인증번호 입력 방식으로 본인인증을 완료하고 기관 검색으로 진입한다.
**화면 흐름:**
```
AUTH-001: 인증 방식 선택
  → AUTH-002: 간편인증서 선택
    → AUTH-005: 간편인증서 목록 로딩
      → AUTH-006: PASS 통신사 선택 (팝업)
        → AUTH-008: 인증 요청
          → AUTH-009: 인증 요청 (대기 중)
            → AUTH-011: 인증번호 입력
              → AUTH-013: 인증 완료
                → SRCH-001: 기관 검색·선택
```
> 진입 경로: MAIN-001 → TERM-001 → AUTH-001
> 변형: 간편인증서 선택 화면은 장차법(AUTH-003)·1열(AUTH-004) 레이아웃 변형 존재.

### SCN-WCK-005: 기관 검색 후 증명서 선택
**분류:** Happy Path
**사용자 목적:** 본인인증을 마친 사용자가 발급 기관을 검색·선택하고 발급할 증명서 종류를 선택한다.
**화면 흐름:**
```
SRCH-001: 기관 검색·선택 (검색어 입력 후 기관 선택)
  → SRCH-002: 증명서 종류 선택
    → ISSU-001: 발급 정보 입력
```
> 진입 경로: AUTH-013(인증 완료) → SRCH-001

### SCN-WCK-006: 발급 정보 입력 → 발급 완료 → 이력 조회
**분류:** Happy Path
**사용자 목적:** 증명서 발급 정보를 입력·확인하고 발급을 완료한 뒤 이력 조회로 이동한다.
**화면 흐름:**
```
ISSU-001: 발급 정보 입력 (필수 항목 입력)
  → ISSU-002: 발급 정보 확인
    → ISSU-003: 발급 처리 중
      → ISSU-004: 발급 완료
        → HIST-001: 발급 이력 목록
```
> 진입 경로: SRCH-002(증명서 선택) → ISSU-001

---

### SCN-WCK-007: 발급 이력 조회 및 상세 확인
**분류:** Happy Path
**사용자 목적:** 발급한 증명서의 이력 목록을 조회하고 개별 발급 건의 상세 내역을 확인한다.
**화면 흐름:**
```
HIST-001: 발급 이력 목록 (이력 항목 탭)
  → HIST-002: 발급 이력 상세
    → (뒤로가기) → HIST-001: 발급 이력 목록
```
> 진입 경로: ISSU-004(발급 완료) → HIST-001, 또는 메인 진입 후 직접 진입.

### SCN-WCK-101: 본인인증 (휴대폰 인증 · 보안문자 방식)

AUTH-001: 인증 방식 선택

      → AUTH-007: 휴대폰 인증 통신사 선택
        → AUTH-008: 인증 요청
          → AUTH-009: 인증 요청 (대기 중)
            → AUTH-010: 보안문자 입력
              → AUTH-013: 인증 완료
```
> 진입 경로: MAIN-001 → TERM-001 → AUTH-001

### SCN-WCK-102: 인증 알림 미수신 → 재요청

AUTH-008: 인증 요청
  → AUTH-009: 인증 요청 (대기 중)

        → AUTH-009: 인증 요청 (대기 중)
          → AUTH-011: 인증번호 입력
```
> 진입 경로: AUTH-006/007(통신사 선택) → AUTH-008

### SCN-WCK-103: 세션 만료 → 메인 재시작
**분류:** 분기
**사용자 목적:** 인증 진행 중 세션이 만료되어 안내를 받고 메인 화면에서 처음부터 다시 시작한다.
**화면 흐름:**
```
AUTH-009: 인증 요청 (대기 중, 세션 만료 발생)
  → AUTH-014: 세션 만료
    → MAIN-001: 위체크 메인 (재시작)
```
> 비고: AUTH-014는 전 구간 어디서든 세션 만료 시 진입 가능. 여기서는 대표적으로 인증 대기 화면을 트리거 지점으로 표기.

### SCN-WCK-104: 발급 실패 → 재시도
**분류:** 분기
**사용자 목적:** 발급 요청이 실패한 사용자가 실패 사유를 확인하고 발급 정보 입력부터 재시도한다.
**화면 흐름:**
```
ISSU-001: 발급 정보 입력
  → ISSU-002: 발급 정보 확인
    → ISSU-003: 발급 처리 중
      → ISSU-005: 발급 실패
        → ISSU-001: 발급 정보 입력 (재시도)
          → ISSU-002: 발급 정보 확인
```
> 진입 경로: SRCH-002(증명서 선택) → ISSU-001

---

### SCN-WCK-105: 발급 실패 → 처음으로
**분류:** 분기
**사용자 목적:** 발급 실패 후 재시도하지 않고 서비스 메인으로 돌아간다.
**화면 흐름:**
```
ISSU-001: 발급 정보 입력
  → ISSU-002: 발급 정보 확인
    → ISSU-003: 발급 처리 중
      → ISSU-005: 발급 실패
        → MAIN-001: 위체크 메인 (처음으로)
```
> 진입 경로: SRCH-002(증명서 선택) → ISSU-001

---

### SCN-WCK-106: 기관 검색 결과 없음
**분류:** 분기
**사용자 목적:** 검색어에 해당하는 기관이 없을 때 안내를 확인하고 검색어를 수정해 재검색한다.
**화면 흐름:**
```
SRCH-001: 기관 검색·선택 (검색어 입력 → 결과 없음 안내 → 검색어 수정 후 재검색)
```
> 진입 경로: AUTH-013(인증 완료) → SRCH-001
> 비고: 검색 결과 없음 상태는 SRCH-001 화면 내 빈 상태(empty state)로 처리되며 별도 화면 없음.

### SCN-WCK-107: 발급 이력 없음 (빈 상태)
**분류:** 분기
**사용자 목적:** 발급 이력이 없는 사용자가 이력 목록에서 빈 상태 안내를 확인한다.
**화면 흐름:**
```
MAIN-001: 위체크 메인 (이력 메뉴 직접 진입)
  → HIST-001: 발급 이력 목록 (빈 상태 안내)
```
> 비고: 빈 상태는 HIST-001 화면 내 empty state로 처리되며 별도 화면 없음.
