## 공통 레이아웃 규칙

> 모든 화면에 예외 없이 적용한다. (출처: `wecheck-spacing-spec.md`)

| 항목 | 값 |
|------|----|
| 화면 너비 | `360px` 고정 |
| 좌우 여백 (Page Margin) | `20px` 고정 |
| 주요 컴포넌트 너비 | `320px` (Button / Input / Dropdown 등) |
| 헤더 합산 높이 | `94px` = `StatusBar`(44px) + `MainHeader`(50px) |
| 콘텐츠 영역 시작점 | `top: 94px`, `padding: 0 20px`, `overflow-y: auto` |
| 하단 버튼 영역 | `padding: 0 20px 20px` — **하단 20px 고정 필수** (single Button 320px / double 155px×2 + gap 10px) |
| 폰 프레임 높이 | `680px` (미리보기 기준, `--screen-height` 토큰) |

> ⚠️ **공통 확인 사항 — 하단 버튼 영역 padding-bottom 필수**
> 버튼이 들어가는 하단 영역(`.bottom`)은 반드시 `padding-bottom: 20px(var(--page-margin))`을 고정 적용한다.
> 버튼 자체가 화면 최하단에 바로 닿지 않도록 아래 여백을 보장해야 한다.
> - single 버튼: `padding: 0 var(--page-margin) var(--page-margin)`
> - double 버튼: `padding: 0 var(--page-margin) var(--page-margin)` (각 버튼 155px, gap 10px)
> - 바텀시트 버튼: `padding: var(--space-md) var(--page-margin) var(--page-margin)` (상단 10px 허용)

**공통 컴포넌트 사용 규칙**
- 상단 상태바: 모든 화면 `<StatusBar />` (360×44px)
- 메인 헤더: `<MainHeader status="default | back | close" />` (360×50px)
- 바텀시트 헤더: `<BottomSheetHeader size="default | small" txtStyle="default | middle" />`
- 단일 CTA: `<Button layout="single" variant="primary | disabled" />`
- 2버튼 세트: `<Button layout="double" position="left | right" />`
- 팝업 하단 버튼: `<Button layout="popup" />` (290px, 하단만 radius 8px)

**Z-Index 레이어 (오버레이/팝업 포함 화면)**

| z-index | 요소 |
|---------|------|
| 1 | 딤 배경 (`top: 94px`부터) |
| 2 | `StatusBar` + `MainHeader` |
| 3 | 1차 바텀시트 |
| 4 / 5 | 2차 딤 / 2차 바텀시트 |
| 20 | 오버레이 팝업 (`OverlayPopup`, 290px 중앙 정렬) |

### AUTH-001 위체크 메인

**파일명:** `A00.위체크메인.html`

**진입 조건**
- 앱 최초 진입 (서비스 시작점)
- 공지가 있을 경우 진입 직후 오버레이 팝업(AUTH-002~004) 노출, 공지가 없으면 그대로 메인 표시

**화면 구성**

| 영역 | 컴포넌트 | 설명 |
|------|----------|------|
| 상단 | `StatusBar` | 360×44px |
| 헤더 | `MainHeader status="default"` | WeCheck 로고 중앙 (back/close 없음) |
| 본문 | 서비스 소개/진입 안내 영역 | 중앙 정렬. 메인 타이틀(`display1` 22px) + 보조 안내(`body2` 14px). 일러스트 또는 브랜드 영역 배치, 상하 gap `24px` |
| 하단 | `Button layout="single" variant="primary"` | "시작하기" (320×50px), 하단 영역 `padding: 0 20px 20px` |

**액션 및 분기**

| 액션 | 조건 | 결과 |
|------|------|------|
| 진입 | 공지 있음 | AUTH-002 / 003 / 004 중 해당 유형 팝업 노출 |
| 진입 | 공지 없음 | 메인 화면 표시 (팝업 없음) |
| "시작하기" 탭 | 정상 | AUTH-005 약관 동의로 이동 |

**오류 처리**

| 케이스 | 메시지 |
|--------|--------|
| 공지 조회 실패 | 팝업 생략하고 메인 정상 노출 (블로킹하지 않음) |
| 서비스 점검 중 | "현재 서비스 점검 중입니다." 안내 후 진행 차단 |
