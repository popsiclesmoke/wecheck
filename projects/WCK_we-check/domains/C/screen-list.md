## AUTH-014 세션 만료 트리거 명세 (CR-002 반영)

세션 만료(AUTH-014)는 인증 진행 중 발생하는 공통 예외 화면이다. "전 구간"이라는 모호한 표현 대신 아래 트리거 조건으로 명세를 확정한다.

| 트리거 구분 | 발생 조건 | 대표 트리거 화면 | 처리 |
|------------|----------|----------------|------|
| 인증 대기 타임아웃 | 인증 요청(AUTH-008) 후 인증 앱 응답을 일정 시간(예: 인증 유효시간) 내 받지 못함 | AUTH-009 (인증 요청 대기 중) | AUTH-014 노출 |
| 입력 타임아웃 | 보안문자/인증번호 입력(AUTH-010·011) 화면에서 유효시간 초과 | AUTH-010, AUTH-011 | AUTH-014 노출 |
| 인증 세션 무효화 | 인증 진행 중 백엔드 세션(인증 토큰) 만료·무효 응답 | AUTH-008~011 (인증 진행 전 구간) | AUTH-014 노출 |

> 비고
> - AUTH-014는 인증 도메인 진행 구간(AUTH-008~011)을 1차 트리거 범위로 한다.
> - 인증 완료(AUTH-013) 이후 후속 도메인(SRCH/ISSU/HIST)에서의 세션 만료·네트워크 오류는 CMN 공통 예외 도메인에서 처리한다. (CR-003 연계)
> - 세션 만료 유효시간 수치는 domain-brief 미확정 사항으로 별도 확정한다.

## 화면 흐름 요약

**시나리오 1: PASS 인증 완료**
```
[AUTH-001] 인증방식선택 → [AUTH-002] 간편인증서선택 → [AUTH-005] 로딩
→ [AUTH-006] PASS통신사선택 → [AUTH-008] 인증요청 → [AUTH-009] 대기
→ [AUTH-011] 인증번호입력 → [AUTH-013] 인증완료 → SRCH 도메인
```

**시나리오 2: 알림 미수신**
```
... → [AUTH-009] 대기 → [AUTH-012] 알림미수신팝업 → [AUTH-008] 재요청
```

**시나리오 3: 세션 만료**
```
[AUTH-008~011] 인증 진행 구간 (타임아웃/세션 무효 발생) → [AUTH-014] 세션만료 → MAIN-001 재시작
```
