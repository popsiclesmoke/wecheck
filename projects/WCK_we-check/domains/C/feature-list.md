## 도메인 개요

서비스 최초 진입 시 공지 팝업을 확인하고, 약관 동의 후 간편인증서(공동인증서) 기반으로
본인인증을 수행한다. 인증 완료 후 후행 도메인(ISSU)으로 진입할 수 있는 상태를 만든다.

## 기능 목록

### [AUTH] Authentication

| Feature ID | 기능명 | 설명 | 인수 조건 | 선행 조건 | 관련 화면 |
|------------|--------|------|-----------|-----------|-----------|
| F-AUTH-001 | 서비스 메인 진입 | 서비스 최초 진입 화면을 표시하고, 공지사항이 있을 경우 팝업을 노출한다. 공지는 1개·스크롤·2개 이상 유형별로 구분된다. | 공지팝업 유형별 정상 노출 및 닫기 동작 | — | A00.위체크메인 / T13·T13_1·T13_2 공지팝업 |
| F-AUTH-002 | 약관 동의 | 서비스 이용약관·개인(신용)정보 수집이용 동의·제3자 제공 동의를 필수/선택 구분하여 받는다. 각 약관 상세 화면을 제공한다. | 필수 약관 전체 동의 시에만 다음 단계 진행 가능 | F-AUTH-001 | A02.위체크메인_약관동의 / Z10.서비스이용약관 / Z12.개인(신용)정보수집및이용동의 / Z13.개인(신용)정보3자제공동의 |
| F-AUTH-003 | 인증 방식 선택 | 본인인증에 사용할 방식을 선택한다. | 인증 방식 선택 후 해당 인증 흐름으로 진입 | F-AUTH-002 | C00.인증방식선택 |
| F-AUTH-004 | 간편인증서 목록 조회 및 선택 | 사용 가능한 간편인증서 목록을 조회하고 사용할 인증서를 선택한다. 목록 로딩 중 로딩 화면을 표시한다. 장차법 접근성 대응 및 1열 레이아웃 변형을 지원한다. | 인증서 선택 시 통신사 선택 단계로 진입. 로딩 중 정상 대기 | F-AUTH-003 | C10.간편인증서_인증서선택 / C10_1.장차법P / C10_2.1열 / C11.로딩 |
| F-AUTH-005 | 통신사 선택 | 선택한 인증서 유형에 따라 PASS 통신사 또는 휴대폰 인증 통신사를 선택한다. PASS는 팝업, 휴대폰 인증은 전용 화면으로 제공된다. | 통신사 선택 후 인증 요청 단계로 진입 | F-AUTH-004 | C11_1.PASS통신사선택_팝업 / C11_2.휴대폰인증통신사선택 |
| F-AUTH-006 | 인증 요청 및 확인 | 선택한 인증서로 인증을 요청하고 결과를 확인한다. 인증 앱 알림 미수신 시 안내 팝업을 제공하며, 보안문자 또는 인증번호 입력을 통해 확인한다. | 인증 성공 시 완료 화면 진입. 각 입력 유형별 정상 처리 | F-AUTH-005 | C12_0·C12_1.인증요청 / C12_2.보안문자입력 / C12_3.인증번호입력 / T12.알림미수신팝업 |
| F-AUTH-007 | 인증 완료 | 본인인증 완료를 안내하고 후행 도메인으로 진입을 제공한다. | 인증 완료 화면 정상 노출 및 다음 단계 이동 | F-AUTH-006 | E00.인증완료 |
| F-AUTH-008 | 세션 만료 처리 | 인증 세션이 만료된 경우 안내하고 재시작 경로를 제공한다. | 세션 만료 시 안내 메시지 노출 및 재시작 동작 | — | E15.세션만료 |
