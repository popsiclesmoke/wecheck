### QA-AUTH-001 인증 방식 선택 — 화면 진입 및 기본 렌더링

**목적**
> TERM 도메인 약관 동의 완료 후 인증 방식 선택 화면(AUTH-001)이 정상 렌더링되는지 확인한다.

**사전 조건**
- TERM 도메인 약관 동의(필수 3종) 완료 상태
- AUTH-001(C00.인증방식선택.html) 화면 진입 직후

**테스트 단계**

| 단계 | 액션 | 기대 결과 |
|------|------|----------|
| 1 | TERM 도메인 약관 동의 완료 후 AUTH-001 화면 진입 | StatusBar(44px), MainHeader(back+로고, 50px) 정상 노출 |
| 2 | 화면 본문 확인 | 타이틀 "본인인증 방식을 선택해주세요" (display1 22px), 인증 방식 카드 목록 정상 노출 |
| 3 | 하단 버튼 상태 확인 | "다음" 버튼이 disabled 상태로 노출 |
| 4 | 공통 레이아웃 확인 | 화면 너비 360px, 좌우 여백 20px, 콘텐츠 영역 top 94px에서 시작 |

**예외 케이스**

| 케이스 | 액션 | 기대 결과 |
|--------|------|----------|
| 인증 방식 목록 조회 실패 | 목록 API 실패 시 | "인증 방식을 불러오지 못했습니다. 다시 시도해주세요." 안내 노출 |

### QA-AUTH-002 인증 방식 선택 — 미선택 시 다음 버튼 비활성

**목적**
> 인증 방식을 선택하지 않은 상태에서 "다음" 버튼이 비활성 상태를 유지하는지 확인한다.

**사전 조건**
- AUTH-001 화면 진입 완료, 인증 방식 미선택 상태

**테스트 단계**

| 단계 | 액션 | 기대 결과 |
|------|------|----------|
| 1 | 인증 방식 카드를 선택하지 않은 상태 유지 | 하단 "다음" 버튼 variant="disabled" 상태 |
| 2 | 비활성 "다음" 버튼 탭 시도 | 버튼 동작 없음, 화면 이동 없음 |

**예외 케이스**

| 케이스 | 액션 | 기대 결과 |
|--------|------|----------|
| — | — | — |

**관련 화면 ID:** AUTH-001
**관련 기능:** F-AUTH-003

**결과**
- [ ] 통과
- [x] 실패 — 사유: C00.인증방식선택.html에서 하단 버튼이 `btn-primary` 클래스로 렌더링되어 있음. 초기 미선택 상태임에도 disabled 스타일이 아닌 primary(활성) 상태로 표시됨. HTML이 "간편인증서" 카드가 selected 상태로 pre-select된 채 렌더링되어 초기 상태(미선택)를 표현하지 않음.

> 수정 필요: 초기 진입 시 어떤 카드도 selected 상태가 아닌 버전으로 HTML 수정 필요. `btn-disabled` 클래스 + `btn-primary` 두 상태를 별도로 표현하거나, 미선택 상태를 기본으로 설정해야 함.

### 실패 항목 상세

#### FAIL-1: QA-AUTH-002 — 인증 방식 선택 초기 상태 오류
**파일:** `C00.인증방식선택.html`
**내용:** 초기 진입 시 "간편인증서" 카드가 `.selected` 상태로 pre-select되어 있고 하단 버튼이 `btn-primary`(활성)으로 렌더링됨. spec 기준 초기 진입 시 미선택 + 버튼 disabled 상태여야 함.
**수정 방향:**
- `div class="method selected"` → `div class="method"` (selected 클래스 제거)
- `<button class="btn btn-primary">다음</button>` → `<button class="btn btn-disabled">다음</button>`
