## 도메인 목적

서비스 이용 전 사용자에게 약관 동의를 받고, 통신사 기반 본인인증으로 실명을 확인한다.
인증 완료 후 발급 가능한 증명서 검색(SRCH 도메인)으로 진입할 수 있는 상태를 만드는 것이 이 도메인의 목적이다.

---

## 주요 기능 범위

- 서비스 이용약관 및 개인정보 수집·이용 동의 (필수/선택 항목 구분)
- 사용자 유형 선택 (개인 / 기업)
- 통신사 선택 (SKT / KT / LGU+ / 알뜰폰)
- 전화번호 입력 및 인증번호 발송
- 인증번호 입력 및 확인
- 주민등록번호 입력 (개인) / 사업자등록번호 입력 (기업)
- 인증 완료 안내
- 인증 실패·오류 처리 (재시도 안내 포함)
- 인증 진행 중 세션 만료 안내 (AUTH-014, 트리거 조건은 screen-list 참조)

---

## 사용자 유형별 분기

| 사용자 유형 | 분기 설명 |
|------------|---------|
| 개인 | 주민등록번호(앞 6자리 + 뒷 1자리) 입력으로 인증 |
| 기업 | 사업자등록번호 입력으로 인증 |

---

## 타 도메인 의존 관계

| 관계 | 도메인 | 설명 |
|------|--------|------|
| 선행 | — | 없음 (서비스 최초 진입 도메인) |
| 후행 | SRCH | 인증 완료 후 기관/증명서 검색 도메인으로 이동 |
| 후행 | MAIN | 세션 만료(AUTH-014) 확인 시 MAIN-001로 재시작 |
| 공유 | 전 도메인 | 인증 세션 정보(CI)를 이후 모든 도메인에서 사용 |
| 공유 | CMN | 인증 완료 후 후속 도메인에서의 세션 만료·네트워크 오류는 CMN 공통 예외 도메인이 처리 |

---

## 미확정 사항

- 기업 사용자 인증 시 추가 입력 항목 여부 (담당자명, 직책 등)
- 약관 항목 수 및 선택 약관 세부 내용
- 인증번호 유효 시간 및 재발송 제한 횟수
- 세션 만료(AUTH-014) 유효시간 수치 및 트리거 정책 — 인증 대기 타임아웃 / 입력 타임아웃 / 세션 무효화의 구체 시간값 확정 필요 (CR-002)
