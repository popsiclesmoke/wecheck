### SCN-SRCH-001 주소 입력 → 검색 결과 선택 → ISSU 진입

**목적**
> SRCH 도메인 기본 흐름 전체가 정상 동작하는지 확인한다.

**사전 조건**
- AUTH 완료 (RSLT-001 인증완료 화면 표시 후 확인)

**테스트 단계**

| 단계 | 액션 | 기대 결과 |
|------|------|----------|
| 1 | SRCH-001 표시 | 주소 입력 화면 표시, 다음 버튼 비활성 |
| 2 | 주소 검색 아이콘 탭 | CMN-008 주소 검색 화면 호출 |
| 3 | 검색어 입력 후 결과 탭 | 주소 자동 입력, SRCH-003 전환, 다음 버튼 활성 |
| 4 | 다음 버튼 탭 | SRCH-005 검색 결과 목록 표시 |
| 5 | 결과 항목 탭 후 선택 완료 | ISSU 도메인 진입 |

**결과**
- [x] 통과
- [ ] 실패 — 사유:

**검수 근거**
- B01: StatusBar + MainHeader(WeCheck 로고 + X 버튼), 주소 검색 입력 필드(플레이스홀더), 검색 아이콘 버튼, 다음 버튼 비활성(color-bg-disabled/cursor:default) 모두 구현 확인
- B02: 우편번호 + 도로명 2행 주소 결과 표시, 다음 버튼 활성(color-primary/cursor:pointer) 구현 확인
- B10: 결과 카드 목록(라디오 버튼 단일 선택, selected 상태 border/radio 강조), "선택 완료" 버튼(활성) 구현 확인

| 3 | 다음 버튼 탭 | SRCH-005 이동 |
| 4 | 결과 선택 완료 | ISSU 도메인 진입 |

**결과**

| 1 | SRCH-001 진입 | 다음 버튼 비활성(disabled) 표시 |
| 2 | 다음 버튼 탭 | 반응 없음 (SRCH-005 이동 안 됨) |

**결과**
- [x] 통과
- [ ] 실패 — 사유:

**검수 근거**
- B01의 `.btn` 스타일: `background: var(--color-bg-disabled); color: var(--color-text-placeholder); cursor: default` 로 시각적 비활성 상태 정확히 구현
- scenario-index 연동 스크립트에서 `cursor === 'default'` 조건으로 다음 화면 이동 차단 처리 확인

### SCN-SRCH-004 선택 변경 확인 — 확인/취소 분기

**목적**
> 결과 목록에서 이미 선택된 항목이 있을 때 다른 항목 탭 시 변경 확인 팝업이 올바르게 동작하는지 확인한다.

**사전 조건**
- SRCH-005에서 항목 1개 선택 완료 상태

**테스트 단계**

| 단계 | 액션 | 기대 결과 |
|------|------|----------|
| 1 | 다른 항목 탭 | SRCH-006 변경 확인 팝업 표시 |
| 2 | 확인 탭 | 팝업 닫힘, 선택 변경됨 |
| 3 | 다시 또 다른 항목 탭 후 취소 탭 | 팝업 닫힘, 기존 선택 유지 |

**결과**
- [ ] 통과
- [x] 실패 — 사유: B11(검색_변경확인.html)이 design-spec SRCH-006 요구사항과 불일치. design-spec은 CMN-002 투버튼 팝업 패턴(딤 배경 + 오버레이 팝업 + 취소/확인 2버튼)을 요구하나, 구현된 B11은 "계좌 변경 신청 확인" 단독 화면(대상자 행 + 기존/변경 계좌 비교 + 단일 "다음" 버튼)으로 구현됨. 팝업 오버레이 구조 없음, 취소 버튼 없음, "선택을 변경하시겠습니까?" 안내 텍스트 없음.

**결함 상세**
| 항목 | 요구사항(design-spec) | 실제 구현(B11) | 판정 |
|------|----------------------|----------------|------|
| 화면 형식 | 딤 배경 + 오버레이 팝업(290px) | 단독 전체 화면 | 불일치 |
| 팝업 안내 문구 | "선택을 변경하시겠습니까?" | 없음 (계좌 변경 신청 확인 내용) | 불일치 |
| 버튼 구성 | 취소(Secondary) + 확인(Primary) 2개 | "다음" 단일 버튼 | 불일치 |
| 화면 내용 | 선택 변경 확인 안내 | 아동수당 계좌 변경 신청 내용 | 불일치 |

### SCN-SRCH-005 공통 주소 검색 연동 확인

**목적**
> CMN 공통 주소 검색 컴포넌트(CMN-008~010)가 SRCH 도메인과 올바르게 연동되는지 확인한다.

**테스트 단계**

| 단계 | 액션 | 기대 결과 |
|------|------|----------|
| 1 | SRCH-001에서 주소 검색 아이콘 탭 | CMN-008 표시 |
| 2 | 존재하지 않는 주소 검색 | CMN-009 결과 없음 표시 |
| 3 | 유효한 주소 검색 후 결과 탭 | SRCH-001/003 화면 복귀, 주소 자동 입력 확인 |

**결과**
- [x] 통과
- [ ] 실패 — 사유:

**검수 근거**
- B01/B01_1: 주소 검색 버튼(search-btn, SVG 돋보기 아이콘) 구현 확인. CMN-008 호출 진입점으로 기능 가능한 구조
- B02/B02_1: 주소 검색 결과 반영 상태(우편번호 "07228" + 도로명 "부산광역시 강서구 …" 2행) 구현 확인. CMN-010 선택 후 자동 입력되는 형태의 address-result 영역 존재
- 정적 프로토타입 특성상 CMN-008~009 화면 자체는 별도 도메인(CMN)에서 관리되며, SRCH 도메인은 연동 진입점(버튼)과 결과 수신 구조(입력 필드)를 올바르게 구현
