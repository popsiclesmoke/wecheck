### SRCH-001 주소 정보 입력

**파일명:** `B01.주소정보입력.html`

**진입 조건**
- RSLT-001 (인증 완료) 확인 버튼 탭 후

**화면 구성**

| 영역 | 컴포넌트 | 설명 |
|------|----------|------|
| 상단 | StatusBar + MainHeader | WeCheck 로고, 닫기(X) 버튼 |
| 페이지 타이틀 | 제목 텍스트 | `heading1` 크기 |
| 본문 | 주소 입력 섹션 | 입력 필드 레이블 + Input(readonly) + 검색 아이콘 버튼 |
| 하단 | Button (Primary, 비활성) | "다음" — 주소 미입력 시 비활성(disabled) |

**액션 및 분기**

| 액션 | 조건 | 결과 |
|------|------|------|
| 주소 검색 아이콘 탭 | — | CMN-008 공통 주소 검색 호출 |
| 주소 선택 완료 (CMN-010) | — | SRCH-003 전환 (주소 입력됨, 버튼 활성) |
| 추가 정보 입력 필요 | — | SRCH-002 전환 |

### SRCH-002 주소 정보 입력 — 추가 항목

**파일명:** `B01_1.주소정보입력_정보추가.html`

**진입 조건**

| 영역 | 컴포넌트 | 설명 |

| 페이지 타이틀 | 제목 텍스트 | |
| 본문 | 주소 입력 섹션 + 추가 항목 섹션 | 추가 입력 항목 표시 (기간, 용도 등) |

### SRCH-005 계좌 변경 대상 목록 선택

**파일명:** `B10.검색_결과목록_선택.html`

> ※ v0.2 갱신: 실제 화면(디자이너)은 일반 "주소 검색 결과"가 아니라 **계좌 변경 대상 목록 선택**(아동수당 지급 목록 조회) 시나리오로 구현되어 있어, 화면 기준으로 명세를 갱신함.

**진입 조건**
- 직전 단계에서 다음 버튼 탭 후

**화면 구성**

| 영역 | 컴포넌트 | 설명 |
|------|----------|------|
| 상단 | StatusBar + MainHeader | WeCheck 로고 + 닫기(X) |
| 스텝 | Step 영역 (고정) | ChartIcon + "2/2" + "아동수당 지급 목록 조회" + 안내 "아래 목록에서 계좌 변경할 항목을 선택해 주세요." |
| 본문 | 결과 카드 목록 (스크롤) | 각 카드: 라디오 + 이름 + 계좌 badge + 은행명·계좌번호(마스킹). 단일 선택 |
| 하단 | Button (Primary) | "다음" |

**액션 및 분기**

| 액션 | 조건 | 결과 |
|------|------|------|
| 카드 탭 | — | 해당 카드 단일 선택(라디오 활성), 기존 선택 해제 |
| 다음 버튼 탭 | 항목 선택됨 | SRCH-006 계좌 변경 확인으로 이동 |

### SRCH-006 계좌 변경 확인

**파일명:** `B11.검색_변경확인.html`

> ※ v0.2 갱신: 실제 화면은 "투버튼 팝업"이 아니라 **계좌 변경 확인 전체 화면**(기존 계좌 → 변경 계좌 비교)으로 구현되어 있어, 화면 기준으로 명세를 갱신함.

**진입 조건**
- SRCH-005에서 변경할 항목 선택 후 다음 버튼 탭 시

**화면 구성**

| 영역 | 컴포넌트 | 설명 |
|------|----------|------|
| 상단 | StatusBar + MainHeader | |
| 대상자 | 대상자 행 | 라벨("대상자") + 구분선 + 이름 |
| 본문 | 계좌 비교 섹션 | 기존 계좌(은행명·계좌번호 마스킹) → chevronDouble 아이콘 → 변경 계좌(강조 표시) |
| 하단 | Button (Primary) | "다음" |

**액션 및 분기**

| 액션 | 조건 | 결과 |
|------|------|------|
| 다음 버튼 탭 | — | 다음 단계(ISSU 도메인 등) 진입 |
