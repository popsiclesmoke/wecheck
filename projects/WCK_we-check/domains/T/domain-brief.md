## 도메인 목적

전 도메인에서 공통으로 발생하는 예외 상황(네트워크 오류, 서버 통신 실패 등)을 일관된 화면으로 안내한다.
product-spec 핵심 기능 8번 "오류·예외 처리" 요건에 대응하며, 특정 도메인에 종속되지 않는 공통 예외 화면을 한곳에서 제공해 SRCH·HIST 등 발급 외 구간의 UX 공백을 해소한다.

---

## 주요 기능 범위

- 공통 네트워크/서버 오류 안내 화면 (CMN-001 예정) — 통신 실패 시 안내 및 재시도/이전 화면 복귀 유도
- (확장 후보) 점검 안내, 공통 로딩 등 도메인 비종속 공통 상태 화면

> 주의: 도메인 고유 예외(AUTH-014 세션 만료, ISSU-005 발급 실패)는 각 도메인이 계속 보유한다. CMN은 그 외 일반화된 공통 예외만 담당한다.

## 미확정 사항

- 공통 오류 화면 종류 세분화 여부 (네트워크 오류 / 서버 오류 / 타임아웃을 단일 화면으로 통합할지, 분리할지)
- 오류 화면에서의 복귀 정책 (직전 화면 재시도 vs. MAIN 복귀) 기본값
- 점검 안내·공통 로딩 화면을 본 도메인 범위에 포함할지 여부

> 본 domain-brief는 CR-003에 따라 도메인 신설·범위 정의까지만 확정한다.
> 실제 화면(CMN-001 등)의 feature-list / screen-list / design-spec / HTML 은 후속 `@planner-agent CMN common` → `@designer-agent` 단계에서 작성한다.
