## 공통 레이아웃 규칙

| 항목 | 값 |
|------|----|
| 화면 너비 | `360px` (`var(--screen-width)`) |
| 좌우 여백 | `20px` (`var(--page-margin)`) |
| 팝업 너비 | `290px` (중앙 정렬, `OverlayPopup`) |
| 팝업 버튼 높이 | `46px` (`var(--button-height)`) |
| 딤 배경 시작 | 헤더 하단(`top: 94px`)부터 |

> CMN 화면은 독립 페이지가 아닌 오버레이/컴포넌트로 동작한다. 전체 화면을 점유하지 않으며 호출 도메인 위에 레이어로 표시된다.

**화면 구성**

### CMN-003 공통 선택 팝업

**파일명:** `T10.(공통)정보입력_선택팝업.html`

**진입 조건**
- 전 도메인 — 목록 중 하나를 선택해야 하는 시점

**화면 구성**

| 영역 | 컴포넌트 | 설명 |
|------|----------|------|
| 딤 배경 | Overlay Dim | 반투명 배경 |
| 팝업 | OverlayPopup (290px) | 중앙 정렬, 헤더 + 리스트 구조 |
| 팝업 헤더 | 타이틀 + X 닫기 | 예: "시도를 선택해 주세요" + 우측 X |
| 본문 | plain 선택 리스트 | 항목별 구분선(border-bottom). 호출 도메인에서 항목 주입 |

> ※ v0.2 갱신: 실제 화면은 라디오/체크 + 하단 "확인" 버튼이 아니라 **헤더(타이틀 + X 닫기) + plain 리스트**이며, 항목 탭 시 즉시 선택·반환된다(하단 확인 버튼 없음). 화면 기준 정정. ※ 강다현 님 재확인 권장

**액션 및 분기**

| 액션 | 조건 | 결과 |
|------|------|------|
| 항목 탭 | — | 선택값 반환, 팝업 닫힘 |
| X 닫기 탭 | — | 팝업 닫힘 (선택 취소) |

### CMN-004 간편인증서 알림 미수신 팝업

**파일명:** `T12.간편인증서_알림미수신팝업.html`

**진입 조건**
- AUTH-009 (인증 대기 중) 화면에서 알림 미수신 시

**화면 구성**

| 영역 | 컴포넌트 | 설명 |
|------|----------|------|
| 상단 | StatusBar + MainHeader | 인증 진행 화면 기저 |
| 본문(기저) | 인증 진행 단계 가이드 | 스텝 "4/5" + 안내, 링크 "기다려도 알림이 오지 않아요!", 하단 버튼 "인증 완료" |
| 딤 배경 | Overlay Dim | 링크 탭 시 팝업 노출 |
| 팝업-제목 | "알림이 오지 않는다면?" | |
| 팝업-본문 | 안내 3종 | "인증수단 APP을 설치해 주세요" / "발급받은 인증서를 선택해 주세요" / "본인 휴대폰 번호인지 확인해 주세요" |
| 팝업-버튼 | Button 2개 (취소·재시도) | 취소: Secondary, 재시도: Primary |

> ※ v0.2 갱신: 실제 화면은 팝업 단독이 아니라 **인증 진행 화면(스텝 4/5 + "인증 완료" 버튼 + "기다려도 알림이 오지 않아요!" 링크) 위에 뜨는 알림 미수신 팝업**이다. 팝업 제목("알림이 오지 않는다면?")·안내 3종 문구 반영. 화면 기준 정정. ※ 강다현 님 재확인 권장

**액션 및 분기**

| 액션 | 조건 | 결과 |
|------|------|------|
| "기다려도 알림이 오지 않아요!" 링크 탭 | — | 알림 미수신 팝업 노출 |
| 재시도 탭 | — | 팝업 닫힘, 인증 요청 재발송 (AUTH-008) |
| 취소 탭 | — | 팝업 닫힘, AUTH-009 유지 |
