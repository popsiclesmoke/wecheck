## 공통 레이아웃 규칙

| 항목 | 값 |
|------|----|
| 화면 너비 | `360px` |
| 좌우 여백 | `20px` |
| 주요 컴포넌트 너비 | `320px` |
| 헤더 합산 높이 | `94px` |
| 버튼 높이 | `50px` |

> RSLT 화면은 완료/오류 결과를 안내하는 최종 화면으로, 일반적으로 콘텐츠 영역에 중앙 정렬 아이콘 + 안내 텍스트 + 버튼 구조를 따른다.

## 화면 설계

---

### RSLT-001 인증 완료

**파일명:** `E00.인증완료.html`

**진입 조건**
- AUTH-011 (인증번호 입력) 성공 후

**화면 구성**

| 영역 | 컴포넌트 | 설명 |
|------|----------|------|
| 상단 | StatusBar + MainHeader | WeCheck 로고, 뒤로가기 버튼 |
| 본문 | 완료 안내 텍스트 | `display1` 크기. "본인인증이 완료되었습니다" 등 |
| 하단 | Button (Primary) | "확인" — 탭 시 SRCH-001 이동 |

### RSLT-004 인증 완료 — 증명서 결과 조회

**화면 구성**

| 영역 | 컴포넌트 | 설명 |
|------|----------|------|
| 상단 | StatusBar + MainHeader | |
| 본문 | 기관 로고 + 발급 내역 목록 | 기관명, 증명서 종류, 발급 결과. 스크롤 가능 |
