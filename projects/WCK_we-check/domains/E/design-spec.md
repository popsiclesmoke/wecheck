## 공통 레이아웃 규칙

| 항목 | 값 |
|------|----|
| 화면 너비 | `360px` |
| 좌우 여백 | `20px` |
| 주요 컴포넌트 너비 | `320px` |
| 헤더 합산 높이 | `94px` |
| 버튼 높이 | `50px` |

> RSLT 화면은 완료/오류 결과를 안내하는 최종 화면으로, 일반적으로 콘텐츠 영역에 중앙 정렬 아이콘 + 안내 텍스트 + 버튼 구조를 따른다.

### RSLT-002 에러 — 미보유 인증서 / 거래 취소 (팝업)

**파일명:** `E02.에러_미보유인증서.html`

**진입 조건**
- AUTH 진행 중 기관 응답 실패·인증서 미보유 등으로 거래가 취소될 때

**화면 구성**

| 영역 | 컴포넌트 | 설명 |
|------|----------|------|
| 상단 | StatusBar + MainHeader(로고만) | 오류 화면 — 뒤로가기 미제공 |
| 배경 | 딤 오버레이 | 단일버튼 팝업(CMN-001 계열) |
| 팝업-본문 | 안내 텍스트 + 오류코드 | "해당 기관에서 응답이 없어 거래를 취소합니다.(HTTP 호출 실패)" / "오류코드: XXXXXX" |
| 팝업-버튼 | Button (Primary) 1개 | "확인" |

> ※ v0.2 갱신: 실제 화면은 본문 오류 아이콘형이 아니라 **딤 오버레이 단일버튼 팝업**(거래 취소·HTTP 호출 실패 안내 + 오류코드)이다. 헤더는 오류 화면 원칙에 따라 **뒤로가기 제거(로고만)**. 화면 기준 정정. ※ 강다현 님 재확인 권장

---

### RSLT-003 에러 — 인증서 오류 (인증 진행 중 실패 팝업)

**파일명:** `E04.에러_인증서오류.html`

**진입 조건**
- 간편인증 진행 중(인증 요청 단계) 인증 요청 실패 시

**화면 구성**

| 영역 | 컴포넌트 | 설명 |
|------|----------|------|
| 상단 | StatusBar + MainHeader(로고만) | 오류 화면 — 뒤로가기 미제공 |
| 본문(기저) | 인증 진행 단계 가이드 | 스텝 "4/5" + 아이콘, 타이틀 "OO인증서 인증 요청", 보조문구 "인증이 완료되면 아래 완료 버튼을 눌러주세요", 링크 "기다려도 알림이 오지 않아요!", 버튼 "인증 완료" |
| 배경 | 딤 오버레이 | 인증 실패 안내 팝업 |
| 팝업-본문 | 안내 텍스트 | "인증 요청에 실패했습니다" / "발급받은 인증서로 재선택 하시거나 종료 후 개인정보를 확인해 주세요" ('재선택' 강조) |
| 팝업-버튼 | Button 2개 (좌 Secondary·우 Primary) | "종료" · "인증서 재선택" |

> ※ v0.2 갱신: 실제 화면은 단순 오류 결과가 아니라 **인증 진행(요청) 화면 위에 뜨는 실패 팝업**(2버튼: 종료/인증서 재선택)이다. screen-list RSLT-003 설명도 이에 맞춰 재대조 권장. 헤더는 뒤로가기 제거(로고만). ※ 강다현 님 재확인 권장
