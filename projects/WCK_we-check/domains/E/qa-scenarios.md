### SCN-RSLT-001 인증 완료 → SRCH 진입

**목적**
> AUTH 완료 후 인증 완료 화면(RSLT-001)이 표시되고 확인 탭 시 SRCH 도메인으로 정상 이동하는지 확인한다.

**사전 조건**
- AUTH-011 인증번호 입력 성공 상태

**테스트 단계**

| 단계 | 액션 | 기대 결과 |
|------|------|----------|
| 1 | AUTH 인증 성공 | RSLT-001 인증 완료 화면 표시 |
| 2 | 완료 문구 확인 | "본인인증이 완료되었습니다" 등 안내 정상 표시 |
| 3 | 확인 버튼 탭 | SRCH-001 이동 |

**결과**
- [ ] 통과
- [x] 실패 — 사유: `E00.인증완료.html` 의 안내 문구가 design-spec 요구사항("본인인증이 완료되었습니다")과 다름. 실제 구현: "완료 버튼을 클릭하고 조회 결과를 확인하세요!". 버튼 라벨도 spec "확인" 과 달리 "완료" 로 표기됨.

**결과**

### SCN-RSLT-003 세션 만료 → MAIN 재시작

**목적**
> 세션 만료 발생 시 세션 만료 화면(RSLT-012)이 표시되고 확인 탭 시 MAIN으로 재시작되는지 확인한다.

**사전 조건**
- AUTH 또는 SRCH/ISSU 진행 중 세션 만료 조건 충족

**테스트 단계**

| 단계 | 액션 | 기대 결과 |
|------|------|----------|
| 1 | 세션 만료 발생 | RSLT-012 세션 만료 화면 표시 |
| 2 | 안내 문구 확인 | 세션 만료 안내 및 재시작 유도 문구 표시 |
| 3 | 확인 버튼 탭 | MAIN-001 이동 (이전 화면 스택 초기화) |

**결과**
- [ ] 통과
- [x] 실패 — 사유: `E15.세션만료.html` 에 2가지 결함 발견. ① 화면 제목이 "인증정보 오류"로 표시되며 design-spec 요구 문구("세션이 만료되었습니다. 처음부터 다시 시작해 주세요.")와 다름. 실제 문구: "요청 세션이 만료되었습니다. 해당 기관 요청 페이지로부터 다시 요청하시기 바랍니다." ② design-spec은 "확인" 단일 버튼(MAIN 이동)을 요구하나, 실제는 "처음으로 돌아가기" 링크 + "재시도" Primary 버튼 구성으로 버튼 역할이 반전됨.

### SCN-RSLT-004 네트워크 오류 → 재시도

**목적**
> 네트워크 오류 발생 시 오류 화면(RSLT-005)이 표시되고 재시도 탭 시 직전 요청이 재시도되는지 확인한다.

**테스트 단계**

| 단계 | 액션 | 기대 결과 |
|------|------|----------|
| 1 | 네트워크 차단 상태에서 API 호출 | RSLT-005 네트워크 오류 화면 표시 |
| 2 | 네트워크 복구 후 재시도 탭 | 직전 요청 재시도, 정상 시 이전 흐름 복귀 |

**결과**
- [ ] 통과
- [x] 실패 — 사유: `E10.네트워크에러.html` 의 버튼 라벨이 "확인"으로 구현되어 있으나, design-spec은 "재시도" 버튼을 요구함. 버튼 라벨 불일치로 사용자에게 재시도 가능성 안내가 누락됨. 화면 구조(네트워크 일러스트 + 오류 안내 텍스트 + Primary 버튼) 자체는 정상.

| 3 | 세션 오류 | RSLT-009 — 세션 오류 안내 표시 |
| 4 | 기타 조회 실패 | RSLT-010 — 조회 실패 안내 표시 |

**결과**
- [x] 통과
- [ ] 실패 — 사유:

> 비고: 4개 파일 모두 오류 아이콘(caution.svg) + 케이스별 안내 텍스트 + 버튼 구조로 정상 구현됨. 각 케이스별 안내 문구 확인 — ① 비회원: "'기관명' 홈페이지 비회원입니다" ② 생년월일: "'생년월일'을 다시 확인해 주세요!" ③ 세션: "세션이 만료되었습니다" ④ 실패: "정보 조회가 실패하였습니다". 4케이스 모두 충분히 구분 가능한 안내 문구로 구현됨.

**목적**
> 2버튼 오류 화면(RSLT-016)에서 재시도와 처음으로 버튼이 각각 올바르게 동작하는지 확인한다.

**테스트 단계**

| 단계 | 액션 | 기대 결과 |
|------|------|----------|
| 1 | 2버튼 오류 화면 표시 | RSLT-016 — 재시도·처음으로 버튼 표시 |
| 2 | 재시도 탭 | 직전 요청 재시도 |
| 3 | 처음으로 탭 | MAIN-001 이동 |

**결과**
- [ ] 통과
- [x] 실패 — 사유: `E40.에러_2버튼.html` 의 버튼 구성이 design-spec과 완전히 다름. spec 요구: "처음으로"(Secondary) + "재시도"(Primary). 실제 구현: "다음"(btn-disabled 비활성) + "건너뛰기"(Primary). 버튼 라벨과 역할이 모두 불일치하며 "처음으로" 버튼이 존재하지 않아 MAIN 복귀 경로가 없음.
