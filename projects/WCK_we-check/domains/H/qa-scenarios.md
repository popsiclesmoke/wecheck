### SCN-HIST-001 발급 이력 목록 조회 및 목록 항목 UI 확인

**목적**
> 발급 이력이 1건 이상 있을 때 목록이 정상 노출되고, 각 항목의 3행 구성(증명서명+상태배지 / 기관명 / 발급일시)이 design-spec에 맞게 구현되어 있는지 확인한다.

**사전 조건**
- MAIN-001 발급이력 진입 메뉴에서 HIST-001(H00.발급이력목록.html) 진입
- 발급 이력 4건 표시 상태

**테스트 단계**

| 단계 | 액션 | 기대 결과 |
|------|------|----------|
| 1 | HIST-001 화면 표시 | 발급 이력 목록 화면 정상 노출, 4건 이력 항목 표시 |
| 2 | 각 항목 1행 확인 | 증명서명(`--font-heading4`) + 상태 배지 우측 정렬 |
| 3 | 각 항목 2행 확인 | 기관명(`--font-body2`, `--color-text-org`) 표시 |
| 4 | 각 항목 3행 확인 | 발급일시(`--font-body4`, `--color-text-disabled`) 표시 |
| 5 | 항목 간 구분선 확인 | 항목 사이 `--color-border-default` 구분선 존재, 마지막 항목 하단 구분선 없음 |
| 6 | 페이지 타이틀 확인 | "발급 이력" 텍스트, `--font-heading1` 적용 |

**관련 화면 ID:** HIST-001
**관련 화면 흐름:** screen-list.md 시나리오 1

### SCN-HIST-002 목록 항목 선택 → 상세 화면 이동 및 상세 정보 확인

**목적**
> 목록에서 발급 건을 선택하면 HIST-003 상세 화면으로 이동하고, 상세 정보(증명서명·발급 기관·발급일시·발급 상태·발급 정보) 5개 항목이 정의 리스트 형태로 정상 노출되는지 확인한다.

**사전 조건**
- HIST-001 발급 이력 목록 표시 상태

**테스트 단계**

| 단계 | 액션 | 기대 결과 |
|------|------|----------|
| 1 | 목록의 첫 번째 항목("주민등록등본") 탭 | HIST-003 발급 이력 상세 화면으로 이동 |
| 2 | 헤더 타이틀 확인 | "발급 이력 상세" 텍스트 표시 |
| 3 | 페이지 타이틀 영역 확인 | 증명서명("주민등록등본") + 상태 배지 가로 배열 표시 |
| 4 | 상세 정보 항목 확인 — 증명서명 | 라벨 "증명서명" / 값 "주민등록등본" |
| 5 | 상세 정보 항목 확인 — 발급 기관 | 라벨 "발급 기관" / 값 "행정안전부" |
| 6 | 상세 정보 항목 확인 — 발급일시 | 라벨 "발급일시" / 값 "2026.06.05 14:32" |
| 7 | 상세 정보 항목 확인 — 발급 상태 | 라벨 "발급 상태" / 값 "발급완료" (`--color-primary` 색상) |
| 8 | 상세 정보 항목 확인 — 발급 정보 | 라벨 "발급 정보" / 값(주소 텍스트) 표시 |
| 9 | 항목 간 구분선 확인 | 각 행 하단 `--color-border-default` 구분선 존재, 마지막 항목 구분선 없음 |
| 10 | 라벨·값 폰트 확인 | 라벨 `--font-body2` / `--color-text-disabled`, 값 `--font-body1` / `--color-text-default` |
| 11 | 하단 CTA 버튼 확인 | 별도 하단 버튼 없음 (spec: "별도 CTA 없음") |

**관련 화면 ID:** HIST-001, HIST-003
**관련 화면 흐름:** screen-list.md 시나리오 1

### SCN-HIST-003 이력 없음(Empty State) 화면 확인

**목적**
> 발급 이력이 0건일 때 HIST-002(이력 없음) 화면이 정상 노출되고, 일러스트·안내 문구·보조 문구가 design-spec에 맞게 구현되어 있는지 확인한다.

**사전 조건**
- MAIN-001 발급이력 진입 메뉴에서 HIST-002(H00_1.발급이력목록_이력없음.html) 진입
- 발급 이력 0건 상태

**테스트 단계**

| 단계 | 액션 | 기대 결과 |
|------|------|----------|
| 1 | HIST-002 화면 표시 | 이력 없음 화면 노출, 리스트 항목 없음 |
| 2 | 헤더 타이틀 확인 | "발급 이력" 텍스트 표시 (HIST-001과 동일) |
| 3 | 페이지 타이틀 확인 | "발급 이력" 텍스트 표시 |
| 4 | Empty State 일러스트 확인 | 일러스트 요소(`data-illust="docCheck"`) 화면 중앙 노출 |
| 5 | 주 안내 문구 확인 | "발급한 이력이 없어요" (`--font-body1` / `--color-text-disabled`) |
| 6 | 보조 안내 문구 확인 | "증명서를 발급하면 이곳에서 확인할 수 있어요" (`--font-body2` / `--color-text-placeholder`) |
| 7 | Empty State 정렬 확인 | 일러스트 + 문구 화면 중앙 정렬 |

**관련 화면 ID:** HIST-002
**관련 화면 흐름:** screen-list.md 시나리오 2

### SCN-HIST-004 상태 배지 컬러 3종 확인

**목적**
> 발급이력 목록 화면에서 발급완료·처리중·발급실패 3가지 상태 배지가 각각 design-spec에 정의된 컬러 토큰으로 구현되어 있는지 확인한다.

**사전 조건**
- HIST-001 발급 이력 목록 표시 상태 (3가지 상태 배지 항목 포함)

**테스트 단계**

| 단계 | 액션 | 기대 결과 |
|------|------|----------|
| 1 | "발급완료" 배지 CSS 클래스 확인 | `.badge-done` 클래스 적용, `--color-primary` 텍스트 색상, `--color-tag-road-bg` 배경 |
| 2 | "처리중" 배지 CSS 클래스 확인 | `.badge-progress` 클래스 적용, `--color-text-disabled` 텍스트 색상, `--color-bg-unselected` 배경 |
| 3 | "발급실패" 배지 CSS 클래스 확인 | `.badge-fail` 클래스 적용, `--color-error` 텍스트 색상, `--color-error-bg` 배경 |
| 4 | 배지 라운드 확인 | `--radius-badge` 토큰 적용 |
| 5 | HIST-003 상세 화면 배지 확인 | 상세 페이지 타이틀 내 배지도 동일 컬러 규칙 적용 |

**관련 화면 ID:** HIST-001, HIST-003
