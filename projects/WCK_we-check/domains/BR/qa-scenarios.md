### SCN-ISSU-001 정상 발급 요청 전체 흐름

**목적**
> ISSU 도메인 기본 흐름(인증번호 입력 → 서류 업로드 → 발급 요청)이 정상 동작하는지 확인한다.

**사전 조건**
- SRCH 완료 (ISSU-001 표시 상태)

**테스트 단계**

| 단계 | 액션 | 기대 결과 |
|------|------|----------|
| 1 | ISSU-001 표시 | 인증번호 입력 화면, 확인 버튼 비활성 |
| 2 | 올바른 인증번호 입력 | 확인 버튼 활성 |
| 3 | 확인 버튼 탭 | ISSU-003 서류 업로드 이동 |
| 4 | 파일 첨부 영역 탭 → ISSU-004 방식 선택 → 파일 선택 | ISSU-003 복귀, 파일 첨부됨 |
| 5 | 필수 서류 모두 첨부 후 발급 요청 탭 | RSLT-014 대기열 로딩 → RSLT-004 결과 조회 |

**결과**
- [ ] 통과
- [x] 실패 — 사유: BR00.인증번호입력.html의 하단 확인 버튼 텍스트가 "다음"으로 구현되어 있으나 design-spec 및 시나리오에서는 "확인"으로 정의됨. 정적 프로토타입 특성상 인증번호 입력 후 버튼 활성화 → 다음 화면 이동 흐름의 화면 간 JS 연동은 scenario-index에서 처리되므로 흐름 구조 자체는 유효하나, 버튼 라벨 불일치로 FAIL 처리.

### SCN-ISSU-003 서류 업로드 방식 선택 및 첨부

**목적**
> 서류 업로드 방식 선택(ISSU-004)이 정상 동작하고 선택한 파일이 첨부되는지 확인한다.

**사전 조건**
- ISSU-003 표시 상태

**테스트 단계**

| 단계 | 액션 | 기대 결과 |
|------|------|----------|
| 1 | 파일 첨부 영역 탭 | ISSU-004 업로드 방식 선택 시트 표시 |
| 2 | 갤러리 선택 탭 | 갤러리 앱 호출 |
| 3 | 파일 선택 완료 | ISSU-003 복귀, 선택한 파일명 표시 |
| 4 | 파일 삭제 버튼 탭 | 파일 제거, 미첨부 상태로 전환 |

**예외 케이스**

| 케이스 | 액션 | 기대 결과 |
|--------|------|----------|
| 방식 선택 취소 | ISSU-004에서 취소 탭 | 시트 닫힘, ISSU-003 유지 |

**결과**
- [ ] 통과
- [x] 실패 — 사유: design-spec(ISSU-004)에는 업로드 방식 항목이 "카메라 촬영 / 갤러리 선택 / 파일 탐색기" 3가지로 정의되어 있으나, BR01_1.파일첨부_업로드선택.html에는 "사진 촬영 / 파일 첨부" 2가지만 구현되어 있음. "갤러리 선택" 항목 누락으로 FAIL 처리.

### SCN-ISSU-004 필수 서류 미업로드 시 발급 요청 버튼 비활성

**목적**
> 필수 서류가 모두 업로드되지 않은 상태에서 발급 요청 버튼이 비활성화되는지 확인한다.

**테스트 단계**

| 단계 | 액션 | 기대 결과 |
|------|------|----------|
| 1 | ISSU-003 진입 | 발급 요청 버튼 비활성(disabled) |
| 2 | 발급 요청 버튼 탭 | 반응 없음 |
| 3 | 필수 서류 전부 첨부 | 발급 요청 버튼 활성화 |

**결과**
- [ ] 통과
- [x] 실패 — 사유: BR01.파일첨부.html이 파일 2건 첨부 완료 상태(활성 버튼)만 구현되어 있고, 필수 서류 미업로드 시의 초기 비활성 상태 화면이 별도 존재하지 않음. screen-list에도 미첨부 초기 상태 화면이 정의되어 있지 않아 "발급 요청 버튼 비활성" 상태를 정적 HTML로 검증 불가. FAIL 처리.
