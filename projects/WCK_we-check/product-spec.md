## 1. 프로젝트 개요

### 1.1 배경 및 목적

**We-Check**는 기업(B2B)을 대상으로 제공하는 **모바일 증명서 발급·제출 서비스**다.
기업이 서비스를 도입하고, 해당 기업의 구성원 또는 고객이 모바일에서 직접 원하는 증명서를 발급받고 결과를 조회한다.

기존에 오프라인 또는 개별 기관 홈페이지를 통해 처리하던 증명서 발급 업무를 단일 모바일 앱으로 통합하여, 발급 절차의 편의성과 효율을 높이는 것이 핵심 목적이다.

### 1.2 타겟 사용자

| 사용자 유형 | 설명 |
|------------|------|
| 개인 사용자 (End User) | 기업이 도입한 We-Check 서비스를 통해 증명서를 발급·조회하는 실사용자 |
| 도입 기업 (B2B 클라이언트) | We-Check를 도입하여 임직원·고객에게 서비스를 제공하는 기업 (앱 설정 범위에 영향) |

### 1.3 핵심 문제 정의

1. 증명서 발급을 위해 기관 방문 또는 복수의 웹사이트를 이용해야 하는 불편함
2. 본인 확인 절차가 분산되어 있어 사용자가 매번 인증을 반복해야 하는 비효율
3. 발급된 증명서의 이력·상태를 한곳에서 확인하기 어려운 문제

## 2. 서비스 범위

### 2.1 포함 범위 (In-Scope)

| 도메인 | 기능명 | 한 줄 설명 |
|--------|--------|-----------|
| AUTH | 약관 동의 | 서비스 이용약관 및 개인정보 수집·이용 동의 |
| AUTH | 본인인증 | 통신사·전화번호·인증번호·주민등록번호 기반 실명 인증 |
| SRCH | 기관 검색·선택 | 발급 가능한 기관 목록 조회 및 선택 |
| SRCH | 증명서 종류 선택 | 선택된 기관의 발급 가능 증명서 목록 조회 및 선택 |
| ISSU | 발급 정보 입력 | 증명서 발급에 필요한 추가 정보 입력 (주소, 기간 등) |
| ISSU | 발급 요청·완료 | 최종 확인 후 발급 요청 및 완료/실패 안내 |
| MAIN | 발급 이력 조회 | 메인에서 직접 진입하는 발급 이력 메뉴 (상세 화면은 현 범위 외 — 추후 정의) |
| RSLT | 결과·오류 안내 | 인증 완료·증명서 결과 조회·각종 오류 및 세션 만료 안내 |
| CMN | 공통 예외 처리 | 공통 팝업·공지·주소검색 및 네트워크 오류 등 전 도메인 공통 예외 |

### 2.2 제외 범위 (Out-of-Scope)

- 기업 관리자 페이지 (도입 기업용 B2B 어드민)
- 증명서 출력·PDF 저장 기능
- 알림(Push) 발송 기능
- 다국어 지원

## 3. 도메인 구성

| 순서 | 도메인 코드 | 도메인명 | 설명 |
|------|-----------|---------|------|
| 1 | MAIN | main | 서비스 메인 진입 화면 + 공지팝업 + 발급이력 진입 메뉴 |
| 2 | TERM | terms | 서비스 이용약관·개인정보 동의 |
| 3 | AUTH | authentication | 인증방식 선택 + 간편인증서 기반 본인인증 전체 프로세스 |
| 4 | SRCH | search | 발급 기관 검색·선택 + 증명서 종류 선택 |
| 5 | ISSU | issuance | 인증번호 입력 + 서류 업로드 + 발급 요청·완료/실패 |
| 6 | RSLT | result | 인증 완료·증명서 결과 조회·각종 오류 및 세션 만료 안내 |
| 7 | CMN | common | 공통 팝업·공지·주소검색 + 네트워크 오류 등 전 도메인 공통 예외 |

> 각 도메인의 상세 범위는 `domains/[DOMAIN]/domain-brief.md` 에서 관리한다.

## 4. 핵심 기능 요약

| 번호 | 기능명 | 설명 |
|------|--------|------|
| 1 | 약관 동의 | 서비스 이용 전 필수·선택 약관에 대한 동의 처리 |
| 2 | 본인인증 | 통신사·전화번호·인증번호·주민등록번호를 통한 실명 인증 |
| 3 | 증명서 발급 기관 선택 | 발급 가능한 기관 목록을 조회하고 선택 |
| 4 | 증명서 종류 선택 | 선택된 기관에서 발급 가능한 증명서 목록 조회 및 선택 |
| 5 | 발급 정보 입력 | 증명서 발급에 필요한 추가 정보 입력 (주소, 기간 등) |
| 6 | 증명서 발급 요청·완료 | 최종 확인 후 발급 요청 및 완료 화면 |
| 7 | 발급 이력 조회 | 과거 발급 이력 목록 및 개별 상세 내역 확인 (메인에서 직접 진입 가능) |
| 8 | 오류·예외 처리 | 네트워크 오류, 인증 실패·세션 만료, 발급 불가 등 상황별 안내 (공통 예외는 CMN 도메인에서 일괄 제공) |

## 6. 제약 사항

- 본인인증은 통신사 PASS 인증 방식에 의존 (외부 API)
- 증명서 발급은 연동 기관의 API 제공 범위 내에서만 가능
- 기업 도입 단위로 서비스가 제공되므로, UI 일부는 도입 기업 설정에 따라 달라질 수 있음
